# 量化交易系统架构重构清理计划

## 📊 系统现状分析

### 文件统计概览
- **总Python文件**: 30,903个
- **archive目录**: 421个Python文件
- **simplified_system**: 363个Python文件
- **根目录src**: 381个Python文件
- **测试文件**: 7,953个
- **报告文件**: 354个Markdown报告

### 核心发现

#### 🏗️ 双重架构问题
1. **根目录src/** - 传统的复杂架构，包含28个子模块
2. **simplified_system/src/** - 简化架构，包含19个专门化子模块
3. 两套架构存在功能重复，造成维护负担

#### 📁 文件分布问题
- **archive/** 占用大量空间但很少使用
- **测试文件**数量过多（7,953个），存在大量重复测试
- **报告文件**冗余（354个），存在时间重复和内容重复
- **配置文件**分散在多个位置

## 🎯 清理目标

### 主要目标
1. **简化架构** - 统一使用simplified_system作为核心架构
2. **减少冗余** - 删除重复和过时的文件
3. **优化性能** - 减少文件索引和加载时间
4. **提升维护性** - 清晰的目录结构和文件组织

### 预期效果
- 文件数量减少60-70%
- 清晰的模块边界
- 更快的开发迭代速度
- 降低新人学习成本

### ❌ 可以清理的文件类别

#### 1. Archive目录（421个Python文件）
- 所有`simplified_system/archive/`中的文件
- 过时的测试脚本和优化器
- 旧版本的回测引擎
- 历史报告和可视化文件

#### 2. 重复测试文件（估计可删除80%）
- 功能相似的测试文件
- 过时的单元测试
- 重复的集成测试

#### 3. 冗余报告文件（估计可删除60%）
- 时间重复的报告（保留最新版本）
- 内容相似的完成报告
- 过时的状态文档

#### 4. 根目录src/复杂架构
- 与simplified_system功能重复的模块
- 过度设计的微服务架构
- 未使用的适配器和网关
