# Task: Implement Authentication and User Management Service

## Description
构建统一的认证和用户管理系统，包括JWT令牌、RBAC权限控制、用户注册登录、密码重置等功能。这将是新架构的核心安全基础。

## Acceptance Criteria
- [ ] JWT认证服务 (登录、刷新、令牌验证)
- [ ] 用户注册和管理API (CRUD操作)
- [ ] 基于角色的权限控制 (RBAC)
- [ ] 密码重置和邮件验证流程
- [ ] API访问频率限制和安全防护
- [ ] 用户会话管理和设备管理
- [ ] 安全审计日志记录

## Technical Details
- **认证框架**: FastAPI + python-jose + passlib
- **权限模型**: RBAC (Role-Based Access Control)
- **数据库**: 统一的用户和权限表
- **安全措施**: 密码哈希、JWT令牌、频率限制
- **邮件服务**: 邮件验证和密码重置

**API端点设计**:
```
POST /auth/register          # 用户注册
POST /auth/login             # 用户登录
POST /auth/refresh           # 刷新令牌
POST /auth/logout            # 用户登出
POST /auth/reset-password    # 重置密码
GET  /users/me               # 获取当前用户信息
PUT  /users/me               # 更新用户信息
GET  /users                  # 管理员获取用户列表
POST /users/{id}/roles       # 分配角色
```

## Dependencies

- [ ] 邮件服务配置 (SMTP)

## Security Considerations
- 使用Argon2id进行密码哈希
- JWT令牌使用RS256签名
- 实施密码强度要求和历史密码检查
- API访问频率限制防止暴力破解
- 敏感操作需要二次验证
- 所有密码相关操作记录审计日志
