# Task: Setup Development Environment and API Gateway

## Description
建立统一的项目开发环境，包括Docker容器化配置、API网关基础框架，以及统一的项目结构。这将为后续所有开发工作提供一致的基础设施。

## Acceptance Criteria
- [ ] 一键启动开发环境脚本 (`docker-compose up -d`)
- [ ] API网关基础框架，支持路由转发和CORS
- [ ] 统一的项目目录结构，清晰的代码组织
- [ ] 开发工具链配置 (ESLint, Prettier, pre-commit hooks)
- [ ] 环境变量配置模板和文档
- [ ] 本地PostgreSQL和Redis容器配置
- [ ] 健康检查端点 (`/health`, `/ready`)

## Technical Details
- **Docker Compose**: 配置开发环境容器网络
- **API Gateway**: 使用FastAPI创建基础路由框架
- **项目结构**: 标准化目录布局 (src/, tests/, docs/)
- **工具链**: ESLint + Prettier + Husky
- **数据库**: PostgreSQL 14 + Redis 6容器

**关键文件**:
```
docker-compose.yml
nginx.conf
gateway/main.py
.env.example
package.json
.eslintrc.js
```

## Dependencies
- [ ] Docker和Docker Compose已安装
- [ ] Node.js 18+ 和 Python 3.10+ 开发环境
- [ ] 现有项目代码备份

## Effort Estimate
- **Size**: M (Medium)
- **Hours**: 16-20小时 (2-3天)
- **Parallel**: true - 可与其他基础设施任务并行

## Definition of Done
- [ ] 开发环境可以一键启动 (`./scripts/dev-start.sh`)
- [ ] API网关响应健康检查 (`curl localhost:8000/health`)
- [ ] 前端开发服务器可以访问 (`localhost:3000`)
- [ ] 数据库连接正常 (`./scripts/db-test.sh`)
- [ ] 代码质量检查工具正常工作
- [ ] 开发文档完整 (README.md + CONTRIBUTING.md)

## Risk Notes
- 确保与现有CBSC系统的端口不冲突
- 数据库迁移工具需要考虑现有数据结构
- 开发环境配置需要兼容CI/CD流程
