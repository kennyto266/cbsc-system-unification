# Task: Implement BacktestAdapter for CBSC Integration

## Description

Create a `BacktestAdapter` class that wraps the existing CBSC backtesting API and provides a clean, notebook-friendly interface for running backtests from Jupyter notebooks. The adapter will handle authentication, job submission, progress tracking, and result retrieval with built-in visualization support.

## Acceptance Criteria

- [ ] BacktestAdapter class with simple run() interface
- [ ] Integration with CBSC backtest API endpoint
- [ ] Progress tracking with real-time updates
- [ ] Result parsing and pandas DataFrame conversion
- [ ] Built-in visualization integration with Plotly
- [ ] Support for parameter sweeps and optimization
- [ ] Error handling and retry logic
- [ ] Async support for non-blocking execution
- [ ] Unit tests with >80% coverage

## Technical Details

### 1. BacktestAdapter Core Class

Create the main adapter class:

```python
class BacktestAdapter:
    """Adapter for CBSC backtesting API with notebook-friendly interface"""

    def __init__(self, api_base_url: str, auth_token: str):
        self.api_base_url = api_base_url
        self.auth_token = auth_token
        self.client = httpx.AsyncClient()

    async def run_backtest(
        self,
        strategy_code: str,
        symbols: List[str],
        start_date: datetime,
        end_date: datetime,
        parameters: Dict[str, Any],
        on_progress: Optional[Callable[[float], None]] = None
    ) -> BacktestResult:
        """Run backtest and return results"""

    async def get_backtest_status(self, job_id: str) -> BacktestStatus:
        """Check status of running backtest"""

    async def cancel_backtest(self, job_id: str) -> bool:
        """Cancel running backtest"""
```
