# Task #67: Strategy Performance Widgets

## Overview

开发和实现策略性能展示组件，为用户提供直观的策略表现、风险指标和投资组合概览。这些组件将作为Dashboard的核心部分，实时展示关键性能指标。

## Technical Requirements

### 组件列表

1. **Strategy Status Widget**
   - 显示策略运行状态（Active/Inactive/Error）
   - 实时状态更新
   - 快速操作按钮（Start/Stop/Pause）

2. **Performance Metrics Widget**
   - 总收益率 (Total Return)
   - 夏普比率 (Sharpe Ratio)
   - 最大回撤 (Max Drawdown)
   - 胜率 (Win Rate)
   - 支持时间范围选择（1D/1W/1M/3M/1Y）

3. **Portfolio Overview Widget**
   - 资产配置饼图
   - 持仓分布
   - 现金余额
   - 总资产价值

4. **Recent Trades Widget**
   - 最近交易列表
   - 交易状态（成功/失败/待处理）
   - 快速筛选和搜索

## Implementation Details

### 文件结构
```
frontend/src/components/Widgets/
├── StrategyStatusWidget.tsx
├── PerformanceMetricsWidget.tsx
├── PortfolioOverviewWidget.tsx
├── RecentTradesWidget.tsx
└── index.ts
```

### 技术栈
- **React 18** with TypeScript
- **Recharts** for data visualization
- **RTK Query** for data fetching
- **WebSocket** for real-time updates

### 数据接口
```typescript
interface StrategyPerformance {
  strategyId: string;
  totalReturn: number;
  sharpeRatio: number;
  maxDrawdown: number;
  winRate: number;
  volatility: number;
  updated_at: string;
}

interface PortfolioData {
  totalValue: number;
  cashBalance: number;
  positions: Array<{
    symbol: string;
    quantity: number;
    value: number;
    percentage: number;
  }>;
}
```

## Acceptance Criteria

1. **响应式设计**
   - 支持桌面端（1200px+）
   - 支持平板端（768px-1199px）
   - 支持移动端（<768px）

2. **实时更新**
   - 数据更新延迟 < 1秒
   - 平滑的动画过渡
   - 错误状态处理

3. **性能要求**
   - 初始渲染时间 < 500ms
   - 内存使用 < 50MB per widget
   - 支持100+策略同时显示

4. **用户体验**
   - Loading状态展示
   - 空状态设计
   - 错误边界处理
   - 键盘导航支持
