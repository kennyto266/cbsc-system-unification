# Task: API接口集成和數據獲取

## Description

實現與現有FastAPI後端的API集成，建立策略數據獲取機制。基於系統中現有的策略管理API端點，開發JavaScript API客戶端，實現策略表現數據、策略清單、狀態控制等核心數據接口的集成，確保前端能夠準確、高效地獲取和更新策略數據。

## Acceptance Criteria
- [ ] 創建API客戶端模塊，封裝HTTP請求和響應處理
- [ ] 集成策略表現數據API (`/api/strategies/performance`)
- [ ] 集成策略清單API (`/api/strategies/list`)
- [ ] 實現策略切換API (`/api/strategies/{strategy_name}/toggle`)
- [ ] 集成策略詳細信息API (`/api/strategies/{strategy_name}/details`)
- [ ] 實現錯誤處理機制，包括網絡錯誤和API錯誤響應
- [ ] 添加數據緩存機制，避免重複請求
- [ ] 實現10秒間隔的自動數據刷新機制
- [ ] API響應時間控制在500ms以內

### API端點集成

#### 策略表現數據
```javascript
// GET /api/strategies/performance
// 返回格式示例：
[
  {
    "name": "DirectRSIStrategy",
    "sharpe_ratio": 1.23,
    "max_drawdown": 0.15,
    "total_return": 0.25,
    "win_rate": 0.65,
    "status": "enabled"
  }
]
```

#### 策略清單管理
```javascript
// GET /api/strategies/list
// POST /api/strategies/{strategy_name}/toggle
// 返回格式示例：
[
  {
    "name": "DirectRSIStrategy",
    "enabled": true,
    "description": "基於牛熊比例的RSI策略"
  }

```

#### 策略詳細信息
```javascript
// GET /api/strategies/{strategy_name}/details
// 返回格式示例：
{

  "last_signal": {"type": "buy", "strength": 75, "timestamp": "2025-12-10T08:30:00Z"}
}
```

### JavaScript實現規範
- **API客戶端**：使用`fetch` API封裝，支持Promise/async-await
- **錯誤處理**：分類處理網絡錯誤、HTTP錯誤、數據格式錯誤
- **緩存策略**：內存緩存，10秒過期時間
- **自動刷新**：使用`setInterval`實現定時更新

### 數據模型定義
```javascript
// 策略表現模型
class StrategyPerformance {
  constructor(name, sharpeRatio, maxDrawdown, totalReturn, winRate, status) {
    this.name = name;
    this.sharpeRatio = sharpeRatio;
    this.maxDrawdown = maxDrawdown;
    this.totalReturn = totalReturn;
    this.winRate = winRate;
    this.status = status;
    this.lastUpdated = new Date();
  }
}

// 策略配置模型
class StrategyConfig {
  constructor(name, enabled, parameters, description) {
    this.name = name;
    this.enabled = enabled;
    this.parameters = parameters;
    this.description = description;
  }
}
```

### 錯誤處理策略
- **網絡錯誤**：自動重試機制(最多3次)
- **HTTP錯誤**：根據狀態碼提供用戶友好的錯誤信息
- **數據錯誤**：數據驗證和默認值處理
- **超時處理**：請求超時(5秒)處理機制
