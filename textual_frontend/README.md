# CBSC 量化交易系统 - Textual TUI 前端

## 简介

CBSC 量化交易系统的终端用户界面（TUI），基于 Textual 框架构建，提供完整的策略管理、回测提交、结果查看和系统配置功能。

## 功能特性

### 1. 策略管理

- ✅ 策略列表查看
- ✅ 新建策略
- ✅ 编辑策略
- ✅ 删除策略
- ✅ 策略参数配置

### 2. 回测管理

- ✅ 提交批量回测任务
- ✅ 配置并行级别（策略级/符号级/参数级/混合模式）
- ✅ 设置进程资源限制
- ✅ 实时监控回测进度
- ✅ 查看回测结果

### 3. 结果查看

- ✅ 回测结果列表
- ✅ 结果详情查看
- ✅ 结果导出（开发中）

### 4. 系统配置

- ✅ API 配置（地址、超时）
- ✅ 界面配置（主题、日志级别）
- ✅ 自动刷新设置

## 安装依赖

```bash
pip install textual httpx
```

或者使用 requirements.txt：

```bash
pip install -r requirements.txt
```

## 运行应用

### 方式 1：使用 Python 模块

```bash
cd textual_frontend
python -m main
```

### 方式 2：使用 run.py

```bash
cd textual_frontend
python run.py
```

### 方式 3：从项目根目录

```bash
python -m textual_frontend.main
```

### 策略管理界面

1. **查看策略列表**：显示所有策略的基本信息（ID、名称、类型、参数、状态）
2. **新建策略**：点击 "新建策略" 按钮，填写策略信息
3. **编辑策略**：选择策略后，可以编辑策略详情
4. **删除策略**：选择策略后，点击 "删除策略" 按钮

**策略字段说明**：

- 策略名称：策略的唯一标识
- 策略类型：策略的类型（如 MA_Crossover_Strategy、RSI_Strategy）
- 日期范围：回测的起止日期
- 参数配置：JSON 格式的策略参数

### 回测提交界面

1. **基本配置**：
   - 策略：选择要回测的策略
   - 代码文件：策略代码文件路径
   - 日期范围：设置回测日期范围

2. **高级配置**：
   - 并行级别：
     - `strategy`：策略级并行（不同策略并行运行）
     - `symbol`：符号级并行（不同交易对并行运行）
     - `parameter`：参数级并行（不同参数组合并行运行）
     - `hybrid`：混合模式（多种并行方式组合）
   - 最大工作进程数：限制并发回测数量
   - 最大内存：限制回测进程内存使用

3. **提交任务**：点击 "提交回测" 按钮提交批量回测任务

### 结果查看界面

1. **查看结果列表**：显示所有回测任务的执行结果
2. **查看详情**：选择结果后，点击 "查看详情" 按钮查看详细信息
3. **导出结果**：选择结果后，可以导出为 JSON 或 CSV 格式（开发中）

**结果字段说明**：

- 任务ID：回测任务唯一标识
- 策略：使用的策略名称
- 开始时间：任务开始时间
- 状态：任务状态（completed、running、failed）
- 收益率：回测收益率
- 夏普比率：风险调整后收益指标

### 进度监控界面

1. **查看任务列表**：显示所有运行中的回测任务
2. **查看进度详情**：选择任务后，显示实时进度信息
3. **自动刷新**：系统会自动刷新任务状态和进度

### 系统配置界面

1. **API 配置**：
   - API 地址：后端 API 服务器地址
   - 超时时间：API 请求超时时间（秒）

2. **界面配置**：
   - 自动刷新：是否自动刷新列表
   - 日志级别：日志输出级别（DEBUG、INFO、WARNING、ERROR）
   - 主题：界面主题（默认、深色、浅色）

## API 集成

系统通过 HTTP API 与后端服务通信，API 客户端支持以下功能：

- `get_strategies()` - 获取策略列表
- `create_strategy()` - 创建策略
- `update_strategy()` - 更新策略
- `delete_strategy()` - 删除策略
- `submit_batch_backtest()` - 提交批量回测
- `get_task_status()` - 查询任务状态
- `get_backtest_results()` - 获取回测结果
- `cancel_backtest()` - 取消回测任务
- `get_health()` - 获取系统健康状态

## 开发指南

### 添加新屏幕

1. 在 `screens/` 目录下创建新的屏幕文件
2. 继承 `Screen` 类
3. 实现 `compose()` 方法构建 UI
4. 在 `main_screen.py` 中添加菜单项

### 添加新功能

1. 在 API 客户端中添加新的 API 方法
2. 在对应的屏幕中添加 UI 组件
3. 实现事件处理逻辑

### 样式定制

编辑 `styles.css` 文件来自定义界面样式。支持以下样式类型：

- 颜色：`$primary`, `$secondary`, `$accent`, `$success`, `$warning`, `$error`
- 文本样式：`bold`, `italic`, `underline`, `strike`
- 对齐：`left`, `center`, `right`
- 边框：`solid`, `dashed`, `dotted`, `double`, `thick`

## 故障排除

### 应用无法启动

1. 检查 Textual 是否正确安装：`pip list | grep textual`
2. 检查 Python 版本（需要 3.8+）
3. 查看终端输出的错误信息

### API 连接失败

1. 检查后端服务是否运行
2. 在系统配置中检查 API 地址设置
3. 检查网络连接

### 界面显示异常

1. 尝试使用默认主题
2. 检查终端是否支持 256 色
3. 重启应用
