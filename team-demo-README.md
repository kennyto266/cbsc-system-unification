## 🔄 演示步骤

1. **初始化项目**
   ```bash
   git clone https://github.com/kennyto266/cbsc-system-unification.git
   cd cbsc-system-unification
   ```

2. **创建功能分支**
   ```bash
   git checkout -b feature/team-demo
   ```

3. **开发功能**
   - 编写代码
   - 运行测试
   - 检查代码质量

4. **提交代码**
   ```bash
   git add .
   git commit -m "feat: Add team collaboration demo"
   ```

5. **推送分支**
   ```bash
   git push -u origin feature/team-demo
   ```

6. **创建 Pull Request**
   - 访问 GitHub
   - 点击 "New Pull Request"
   - 填写 PR 模板
   - 指定审查者

7. **代码审查**
   - 团队成员审查代码
   - 提供反馈和建议
   - 修改和完善

8. **合并代码**
   - 通过所有审查
   - 合并到主分支
   - 删除功能分支

## 🎉 最佳实践

1. **分支命名规范**
   - `feature/功能描述`
   - `fix/问题描述`
   - `hotfix/紧急修复`

2. **提交消息格式**
   ```
   type(scope): description
   
   feat: 新功能
   fix: 错误修复
   docs: 文档更新
   ```

3. **代码审查原则**
   - 建设性反馈
   - 及时响应
   - 质量优先
