# CBSC量化交易系統生產環境監控部署方案

## 1. 系統概覽

基於現有系統分析，CBSC量化交易系統包含以下核心組件：
- **FastAPI後端服務** (端口3004) - 核心API服務
- **WebSocket實時數據** - 策略數據推送
- **PostgreSQL數據庫** - 數據存儲
- **Redis緩存** - 數據緩存
- **React前端儀表板** - 用戶界面
- **477技術指標計算** - 核心業務邏輯

## 4. 告警策略

### 4.1 嚴重級別定義
- **Critical (P0)**: 系統不可用，影響所有用戶
- **Warning (P1)**: 性能下降，影響部分用戶
- **Info (P2)**: 需要關注的潛在問題

### 4.2 告警規則
```yaml
# 服務不可用
- alert: ServiceDown
  expr: up == 0
  for: 1m
  labels:
    severity: critical
  annotations:
    summary: "服務 {{ $labels.instance }} 已宕機"

# 高錯誤率
- alert: HighErrorRate
  expr: rate(http_requests_total{status_code=~"5.."}[5m]) > 0.05
  for: 5m
  labels:
    severity: warning
  annotations:
    summary: "錯誤率超過5%"

# 高響應時間
- alert: HighResponseTime
  expr: histogram_quantile(0.95, rate(http_request_duration_seconds_bucket[5m])) > 0.5
  for: 5m
  labels:
    severity: warning
  annotations:
    summary: "95%請求響應時間超過500ms"
```

## 6. 部署配置

### 6.1 基礎設施要求
- **CPU**: 4核心以上
- **內存**: 16GB以上
- **存儲**: 200GB SSD
- **網絡**: 1Gbps帶寬

### 6.2 服務端口分配
```yaml
Prometheus:    9090
Grafana:       3010
AlertManager:  9093
Loki:          3100
Node Exporter: 9100
cAdvisor:      8080
Blackbox:      9115
```

### 6.3 數據保留策略
- **Prometheus**: 30天原始數據，90天降採樣數據
- **Loki**: 15天日誌數據
- **Grafana**: 無限制儀表板配置
