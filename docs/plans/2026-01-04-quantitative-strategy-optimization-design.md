# 高 SR-MDD 量化交易策略優化系統設計

> 設計日期: 2026-01-04
> 目標: 個人使用、不公開、找出高夏普比率低最大回撤的股票量化策略

---

## 1. 系統概述

個人使用的股票多策略量化優化系統，採用純數據驅動方法，目標是自動發現並維護高夏普比率（SR）、低最大回撤（MDD）的交易策略組合。

**核心特點：**
- 多時間框架並行優化（分鐘級/日級/週級）
- 多維數據融合（價格+基本面+另類數據）
- 自適應參數優化引擎
- 多策略動態組合管理
- 嚴格回測驗證流程
- 實時風控監控系統
- 完全本地化部署，保護策略隱私

---

## 2. 系統架構

```
數據層 → 特徵工程 → 多策略優化引擎 → 組合構建 → 回測驗證 → 實盤監控
```

### 2.1 多時間框架並行處理
- **分鐘級管道**：1分/5分/15分 K線，日內策略
- **日級管道**：OHLCV + 成交量，波段策略（數天到數週）
- **週級管道**：週 OHLCV，長期策略（數月）

### 2.2 多維數據融合層
- **價格數據**：Yahoo Finance API、HKEX 數據庫
- **基本面數據**：財報、估值指標、宏觀經濟
- **另類數據**：新聞情緒、社交媒體、搜尋熱度

### 2.3 自適應優化引擎
- 貝葉斯優化或遺傳算法
- 動態調整參數以最大化 SR/MDD 綜合分數

### 2.4 策略組合模塊
- 4-6個子策略並行運行
- 動態權重分配
- 風控層實時計算

## 3. 自適應優化引擎

### 3.1 目標函數

```
綜合分數 = α × 夏普比率 - β × 最大回懲率 + γ × 卡瑪比率
```

默認權重：α=0.5, β=0.3, γ=0.2（可自定義）

### 3.2 優化算法

**主算法：貝葉斯優化**
- 高效搜索高維參數空間
- 特別適合昂貴的回測評估
- 採用 Gaussian Process 作為代理模型

**備選：遺傳算法 + 粒子群混合**
- 全局搜索能力強
- 避免陷入局部最優

**加速：並行回測**
- 利用多核 CPU 同時評估多組參數

### 3.3 動態適應機制

**滑動窗口優化**
- 每30天重新優化參數
- 適應市場制度變化

**在線學習**
- 實盤數據持續更新模型
- 避免過度擬合歷史數據

**市場狀態識別**
- 牛市/熊市/震盪市區分
- 不同市況使用不同參數組

### 3.4 參數空間定義

**趨勢策略參數**
- MA 週期組合
- 突破閾值
- 止損止盈幅度

**均值回歸策略參數**
- Z-score 閾值
- 持有週期
- 相關性窗口

**動量策略參數**
- 排名週期
- 弱勢篩選標準
- 換手率控制

**風控參數**
- 單倉上限
- 組合目標波動率
- 相關性限制

### 3.5 避免過擬合

**Walk-Forward 分析**
- 訓練-驗證-測試滾動窗口
- 樣本外測試驗證

**交叉驗證**
- K-fold 時序交叉驗證
- 時間序列交叉驗證

**正則化懲罰**
- 參數複雜度懲罰項
- L1/L2 正則化

## 4. 多策略組合構建

### 4.1 子策略定義

**趨勢跟蹤策略**
- 雙移動平均交叉
- 布林帶突破
- 唐奇安通道突破

**均值回歸策略**
- 統計套利
- 配對交易
- RSI 均值回歸

**動量策略**
- 相對強弱策略
- 基本面篩選動量
- 盈餘動量

**波動率策略**
- ATR 突破
- 歷史波動率均值回歸

**多因子模型**
- Fama-French 五因子擴展
- 動態因子選擇

### 4.2 動態權重分配

**常規方法**
- 等權分配
- 風險平價（Risk Parity）
- Kelly 公式

**高級方法**
- 黑-利特曼模型（Black-Litterman）
- 馬可維茨均值-方差優化
- 層級風險平價（Hierarchical Risk Parity）

**自適應調整**
- 根據最近表現動態調整權重
- 表現好的策略逐漸加權
- 表現差的策略逐漸減權

### 4.3 相關性控制

- 計算策略間相關性矩陣
- 避免高相關性策略重疊
- 設定最大相關性閾值（如 0.7）
- 使用聚類分析選擇低相關性策略組合

### 4.4 再平衡機制

**時間觸發**
- 每週再平衡
- 每月再平衡

**波動觸發**
- 權重漂移超過 ±5% 時再平衡

**成本控制**
- 設定最低再平衡閾值
- 避免過度交易產生高手續費

### 4.5 組合層風控

- 目標年化波動率：10-15%
- 單策略最大權重：30%
- 行業暴露限制
- 因子暴露限制
- 最大集中度限制

## 6. 回測與驗證系統

### 6.1 回測引擎架構

**基於現有 VectorBT 引擎**
- 高性能向量化回測
- 支援多時間框架並行回測
- 已集成到系統中

**交易成本模擬**
- 手續費：萬分之五
- 滑點：買入 +0.1%、賣出 -0.1%
- 衝擊成本：大額交易影響
- 融資融券成本

**多資產組合回測**
- 支援同時回測多個資產
- 自動計算組合指標
- 考慮資產間相關性

### 6.2 回測指標計算

**收益指標**
- 年化收益率
- 累計收益率
- 月度收益分布
- 正收益月份占比

**風險指標**
- 年化波動率
- 最大回撤（MDD）
- 最大回撤持續期
- 下行風險
- Value at Risk (VaR)
- Conditional VaR (CVaR)

**風險調整指標**
- 夏普比率（SR）
- 索提諾比率
- 卡瑪比率
- 信息比率
- 歐米伽比率

**統計指標**
- 勝率
- 盈虧比
- 最大連續盈利次數
- 最大連續虧損次數
- 交易頻率
- 平均持有期

### 6.3 嚴格驗證流程

**樣本內測試**
- 使用 70% 歷史數據訓練參數
- 找出最優參數組合

**樣本外測試**
- 使用剩餘 30% 數據驗證
- 檢測過擬合
- 確保策略泛化能力

**Walk-Forward 分析**
- 滾動窗口優化
- 模擬實盤參數更新
- 更真實的預測能力評估

**蒙特卡羅模擬**
- 1000 次隨機路徑模擬
- 檢驗策略穩定性
- 計算收益分置信區間

**排列測試**
- 隨機打亂時間序列
- 檢驗策略是否捕捉真實模式
- 排除運氣成分

### 6.4 壓力測試

**極端市場條件**
- 2008 金融危機
- 2020 疫情暴跌
- 2000 科網泡沫

**黑天鵝事件**
- 單日暴跌 >7% 的表現
- 極端波動率時期的表現

**流動性危機**
- 成交量極低時的表現
- 买卖價差擴大的影響

**高相關性崩潰**
- 所有資產同時下跌時的表現
- 分散化失效時的保護

### 6.5 實盤模擬

**Paper Trading 模式**
- 實時信號模擬交易
- 不實際下單
- 驗證策略實際可行性

**模擬帳戶追蹤**
- 記錄所有模擬交易
- 追蹤模擬收益和回撤
- 與回測對比分析

**實盤對比**
- Paper Trading 表現 vs 回測預期
- 檢測策略衰減
- 調整參數

## 7. 實盤部署與監控

### 7.1 部署架構

**階段 1：模擬盤模式（初期）**
- Paper Trading 驗證 1-3 個月
- 無實際資金風險
- 驗證系統穩定性

**階段 2：小資金試運行**
- 資金的 5-10% 實盤測試
- 驗證實際交易執行
- 觀察滑點和成本影響

**階段 3：全資金部署**
- 確認穩定後全面啟動
- 持續監控和優化

**現有集成**
- 已整合 Futu 牛牛 API
- 可直接下單執行交易

### 7.2 實時監控儀表板

**策略狀態監控**
- 運行中 / 暫停 / 錯誤狀態
- 各策略實時表現
- 系統健康狀況

**實時績效監控**
- 當日收益 / 當月收益 / 當年收益
- 實時夏普比率
- 當前最大回撤
- 與目標對比

**持倉監控**
- 各策略持倉明細
- 行業暴露分布
- 風險敞口
- 現金狀況

**告警系統**
- 回撤超標告警（>15%）
- 單日損失告警（>-3%）
- 異常交易告警
- 系統故障告警
- 數據延遲告警

### 7.3 自動化風控

**實時回撤監控**
- MDD 超過預設值（如 15%）自動減倉 50%
- MDD 超過 20% 暫停所有新開倉
- 逐步恢復倉位

**單日損失限制**
- 單日虧損 >3% 暫停新開倉
- 單日虧損 >5% 減倉 50%
- 單日虧損 >7% 全部平倉

**波動率異常檢測**
- VIX 暴漲（>30）降低倉位
- 波動率突增時切換防守策略

**相關性崩潰保護**
- 市場恐慌時切換現金/國債
- 降低策略相關性

### 7.4 策略維護

**自動再優化**
- 每 30 天重新運行優化引擎
- 更新策略參數
- 平滑過渡避免劇烈變化

**性能衰減檢測**
- 實盤 SR 低於回測 80% 時發出警報
- 自動觸發重新優化
- 必要時替換策略

**策略替換機制**
- 表現差的策略自動降權
- 連續 3 個月負收益的策略移除
- 新策略入場測試

**A/B 測試**
- 新策略與舊策略並行運行
- 對比實際表現
- 優者勝出

### 7.5 個人使用隱私保護

**本地化部署**
- 所有數據本地存儲
- 不上傳雲端
- 完全掌控

**無用戶系統**
- 無需賬戶登錄
- 本地直接訪問
- 減少攻擊面

**無分享功能**
- 策略完全保密
- 不對外分享
- 保護競爭優勢

**可選數據加密**
- 敏感數據加密存儲

## 10. 風險提示

本系統僅供個人研究使用，不構成投資建議。量化交易存在以下風險：

- **過擬合風險**：歷史表現不代表未來收益
- **市場變化風險**：市場制度變化可能導致策略失效
- **技術風險**：系統故障、數據錯誤、網絡問題
- **執行風險**：滑點、衝擊成本、流動性不足
- **極端事件**：黑天鵝事件可能導致超預期虧損

請謹慎管理風險，僅使用可承受損失的資金進行交易。
