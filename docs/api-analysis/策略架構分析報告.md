## 2. 架構問題分析

### 2.1 職責不清

1. **權限控制分散**
   - `cbsc_strategy_api.py` 和 `personal_strategy_endpoints.py` 都有權限邏輯
   - 權限驗證方式不一致

2. **業務邏輯重疊**
   - 三個文件都實現了策略管理功能
   - 沒有清晰的邊界定義

3. **數據存儲混亂**
   - 每個管理器都有自己的存儲結構
   - 數據同步問題潛在風險

### 2.2 可維護性問題

1. **代碼重複率高**
   - 總重複代碼約700行，佔總代碼量25%
   - 修改一個功能需要在多個文件中同步

2. **測試覆蓋困難**
   - 重複的邏輯需要重複測試
   - 邊界條件測試複雜

3. **版本同步風險**
   - 功能更新容易遺漏某個文件
   - API版本不一致風險

### 2.3 性能問題

1. **重複初始化**
   - 多個管理器實例可能同時存在
   - 內存使用效率低

2. **數據冗餘**
   - 相同的數據在多個地方存儲
   - 數據一致性難以保證

3. **資源競爭**
   - 同一資源可能被多個管理器訪問
   - 並發控制複雜

## 3. 新模組化架構設計

### 3.1 設計原則

1. **單一職責原則 (SRP)**
   - 每個模組只負責一個特定功能
   - 明確的接口定義

2. **開放封閉原則 (OCP)**
   - 易於擴展新功能
   - 修改時不影響現有代碼

3. **依賴倒置原則 (DIP)**
   - 高層模組不依賴低層模組
   - 使用抽象接口

4. **接口隔離原則 (ISP)**
   - 客戶端不應該依賴它不需要的接口
   - 細粒度的接口設計

### 3.3 核心模組設計

#### 3.3.1 基礎服務層 (base.py)

```python
from abc import ABC, abstractmethod
from typing import TypeVar, Generic, Optional, List, Dict

T = TypeVar('T')

class AbstractBaseService(ABC, Generic[T]):
    """抽象基礎服務類"""

    @abstractmethod
    async def create(self, data: dict) -> T:
        """創建實體"""
        pass

    @abstractmethod
    async def get(self, id: str) -> Optional[T]:
        """獲取實體"""
        pass

    @abstractmethod
    async def update(self, id: str, data: dict) -> T:
        """更新實體"""
        pass

    @abstractmethod
    async def delete(self, id: str) -> bool:
        """刪除實體"""
        pass

    @abstractmethod
    async def list(self, filters: dict = None, pagination: dict = None) -> List[T]:
        """列表查詢"""
        pass
```

#### 3.3.2 策略服務層 (services/strategy_service.py)

```python
class StrategyService(AbstractBaseService[Strategy]):
    """策略管理核心服務"""

    def __init__(
        self,
        repository: StrategyRepository,
        performance_service: PerformanceService,
        template_service: TemplateService
    ):
        self.repository = repository
        self.performance_service = performance_service
        self.template_service = template_service

    async def create_strategy(self, request: CreateStrategyRequest, user_id: int) -> Strategy:
        """創建策略 - 統一的創建邏輯"""
        # 1. 驗證請求
        await self._validate_create_request(request)

        # 2. 生成策略ID
        strategy_id = await self._generate_strategy_id(request, user_id)

        # 3. 應用模板（如果有）
        parameters = await self._apply_template(request)

        # 4. 創建策略實例
        strategy = Strategy(
            id=strategy_id,
            name=request.name,
            description=request.description,
            strategy_type=request.strategy_type,
            parameters=parameters,
            user_id=user_id
        )

        # 5. 持久化
        await self.repository.save(strategy)

        # 6. 初始化性能追蹤
        await self.performance_service.initialize_tracking(strategy_id)

        return strategy
```

#### 3.3.3 個人化服務層 (personal.py)

```python
class PersonalStrategyService:
    """個人策略服務 - 依賴策略服務"""

    def __init__(
        self,
        strategy_service: StrategyService,
        execution_service: ExecutionService,
        notification_service: NotificationService
    ):
        self.strategy_service = strategy_service
        self.execution_service = execution_service
        self.notification_service = notification_service
        self.websocket_manager = WebSocketManager()

    async def get_user_dashboard(self, user_id: int) -> PersonalDashboardData:
        """獲取個人儀表板數據"""
        # 使用策略服務獲取數據
        strategies = await self.strategy_service.list_by_user(user_id)

        # 聚合數據
        dashboard_data = await self._aggregate_dashboard_data(strategies)

        # 設置WebSocket更新
        await self.websocket_manager.register_user(user_id)

        return dashboard_data
```
