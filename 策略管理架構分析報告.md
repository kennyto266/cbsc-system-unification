# CBSC量化交易策略管理系统架构分析报告

## 执行摘要

本报告对CBSC量化交易策略管理系统进行了全面的架构分析。该系统是一个企业级的量化交易平台，采用微服务架构，整合了现代化的前端和后端技术栈，提供完整的策略开发、执行、监控和管理功能。

## 1. 项目概览

### 1.1 系统概述
- **项目名称**: CBSC量化交易策略管理系统 (Computer-Based Strategy Control System)
- **核心功能**: 量化交易策略管理、实时监控、回测分析、风险控制
- **技术架构**: 前后端分离的微服务架构
- **部署方式**: Docker容器化部署

### 1.2 技术栈
- **前端**: React 18 + TypeScript + Vite + Tailwind CSS
- **后端**: FastAPI + Python 3.13
- **数据库**: PostgreSQL 15
- **缓存**: Redis 7
- **实时通信**: WebSocket
- **容器化**: Docker + Docker Compose
- **测试框架**: Jest + Pytest

## 3. 数据库设计

### 3.1 数据模型
采用Pydantic进行数据验证和序列化，定义了完整的领域模型：

#### 3.1.1 核心模型
```python
# 市场数据模型
class MarketData(BaseModel):
    symbol: str
    timestamp: datetime
    open_price: Decimal
    high_price: Decimal
    low_price: Decimal
    close_price: Decimal
    volume: int

# 交易信号模型
class TradingSignal(BaseModel):
    symbol: str
    signal_type: SignalType
    strength: float
    timestamp: datetime
    price: Decimal

# 投资组合模型
class Portfolio(BaseModel):
    holdings: List[Holding]
    cash: Decimal
    total_value: Optional[Decimal]
```

#### 3.1.2 数据库表结构
- **用户表**: 用户基础信息、权限管理
- **策略表**: 策略配置、参数、状态
- **执行记录表**: 策略执行历史、性能指标
- **市场数据表**: 实时和历史市场数据
- **日志表**: 系统操作日志、审计记录

### 3.2 缓存策略
采用多级缓存架构：
- **L1缓存**: 内存缓存 (TTLCache, LRUCache)
- **L2缓存**: Redis缓存 (持久化、分布式)
- **缓存策略**: 支持多种淘汰策略和TTL管理
