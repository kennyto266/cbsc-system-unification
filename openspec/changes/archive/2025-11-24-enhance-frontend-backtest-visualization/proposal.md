# 增强前端回测可视化 - Equity Curve 功能

## Why

### 当前痛点
港股量化交易系统具备强大的后端回测引擎（Rust核心 + Python API），但前端可视化功能存在明显不足：

1. **用户体验不佳**: `frontend/src/views/Backtest.vue` 仅有占位内容，无法完成完整的回测流程
2. **数据展示不直观**: 缺少专业的Equity Curve图表，用户难以直观评估策略表现
3. **功能割裂**: 现有 `BacktestComparison.vue` 组件功能单一，无法满足深度分析需求
4. **效率低下**: 用户无法通过界面完成参数配置、结果分析、报告导出的完整工作流

### 业务价值
通过增强前端可视化功能，预期带来以下价值：
- **提升用户效率**: 完成回测时间从30分钟缩短到5分钟
- **改善决策质量**: 可视化分析帮助用户更好地理解策略表现
- **增强产品竞争力**: 专业级前端体验对标行业领先产品
- **降低使用门槛**: 图形化界面降低量化分析的专业要求

### 技术必要性
- **发挥后端优势**: 现有Rust核心引擎性能卓越，需要匹配的前端展示
- **完善技术栈**: Vue 3 + ECharts技术栈已具备，需要专业化应用
- **数据闭环**: 从数据采集到结果展示的完整链路

## 变更概述

为港股量化交易系统添加专业的前端回测可视化功能，重点关注回测走势图表和Equity Curve（资金曲线）功能，为用户提供直观的策略表现分析工具。

## 问题背景

当前系统具备完整的后端回测引擎（Rust核心 + Python API），但前端回测可视化功能较为基础：

1. **回测页面**: `frontend/src/views/Backtest.vue` 仅有占位内容
2. **图表组件**: 虽然有 `BacktestComparison.vue`，但缺乏完整的回测结果展示
3. **Equity Curve**: 缺少专门的资金曲线图表和分析功能
4. **数据接口**: 后端已有回测API，但前端集成不完整

## 建议解决方案

### 核心功能
1. **完整的回测界面**: 替换占位页面为功能完整的回测工具
2. **Equity Curve图表**: 专业的资金曲线可视化，支持多策略对比
3. **回测走势图表**: 价格走势、买卖信号、收益分布等可视化
4. **性能指标仪表板**: 夏普比率、最大回撤、胜率等关键指标展示
5. **交互式分析**: 参数调整、时间段选择、策略对比等功能

### 技术实现
- **前端**: Vue 3 + TypeScript + ECharts，复用现有图表组件架构
- **后端**: 集成现有回测API (`/api/custom/tests/backtest`)
- **数据格式**: 标准化回测结果数据结构，支持Equity Curve数据

### 包含内容

- ✅ Equity Curve专业图表组件

3. **用户体验**: 界面直观易用，加载速度快，交互流畅
4. **数据准确性**: 前端展示数据与后端计算结果完全一致

### 依赖项
- 现有回测API稳定性 (`/api/custom/tests/backtest`)
- 前端图表库 (`echarts`, `vue-echarts`)
- 数据模型定义一致性
