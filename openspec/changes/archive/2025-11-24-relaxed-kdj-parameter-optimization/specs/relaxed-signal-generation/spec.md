# 放寬信號生成系統規格

## Requirements

### ADDED Requirement: 彈性KDJ信號生成器

#### Description
實現放寬的KDJ信號生成邏輯，支持多層次信號條件，提高信號生成頻率。

#### Scenario: 多層次信號條件
當KDJ指標在不同區間時，系統應該生成不同強度的交易信號：
- 強烈信號：K>70且D>65（賣出），K<30且D<35（買入）
- 中等信號：K>60且J>D（賣出），K<40且J<D（買入）
- 微弱信號：|K-D|>10（根據KD線方向）
- 中性：其他情況

#### Acceptance Criteria
- [ ] 每個KDJ策略至少生成5個交易信號
- [ ] 支持信號強度區分（-1, -0.5, 0.5, 1, 0）
- [ ] 保持與原始系統的API兼容性

### ADDED Requirement: 自適應閾值調整

#### Description
根據市場條件動態調整KDJ進場閾值，避免在平穩市場中無法生成信號的問題。

#### Scenario: 市場波動性適應
當市場波動性較低時，系統應該自動降低進場閾值；當波動性較高時，使用相對較高的閾值以減少假信號。

#### Acceptance Criteria
- [ ] 實現波動性計算（基於歷史價格標準差）
- [ ] 閾值根據波動性動態調整（±20%範圍）
- [ ] 記錄閾值調整歷史便於分析

### ADDED Requirement: 多策略信號融合

#### Description
結合多種KDJ信號生成策略，提供更豐富的交易信號選擇。

#### Scenario: 四種信號策略並存
系統同時提供經典KDJ、趨勢KDJ、動量KDJ和組合KDJ四種策略，用戶可以選擇使用或組合使用。

#### Acceptance Criteria
- [ ] 實現4種不同的KDJ信號生成策略
- [ ] 支持策略組合和權重配置
- [ ] 提供策略表現對比分析

### MODIFIED Requirement: KDJ指標計算引擎

#### Description
增強現有KDJ計算引擎，支持更靈活的參數配置和性能優化。

#### Scenario: 大規模參數計算
當需要計算600個不同KDJ參數組合時，系統應該能夠高效處理，計算時間控制在合理範圍內。

#### Acceptance Criteria
- [ ] 支持5-300範圍K週期，1-20範圍D週期
- [ ] 實現向量化計算提高性能
- [ ] 計算結果精度與原始系統保持一致

### MODIFIED Requirement: 信號驗證機制

#### Description
增強信號驗證邏輯，確保放寬條件下的信號質量。

#### Scenario: 信號合理性檢查
生成的交易信號需要通過合理性檢查，包括：
- 信號分佈合理性（不應過度集中）
- 時間間隔合理性（避免過於頻繁交易）
- 連續同向信號檢查

#### Acceptance Criteria
- [ ] 實現多維度信號驗證
- [ ] 提供信號質量評分機制
- [ ] 記录驗證失敗的信號和原因
