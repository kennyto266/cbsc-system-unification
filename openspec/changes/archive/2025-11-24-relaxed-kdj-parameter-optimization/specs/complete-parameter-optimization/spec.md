# 完整參數優化系統規格

## Requirements

### ADDED Requirement: 參數網格生成器

#### Description
生成完整的KDJ參數組合網格，覆蓋0-300範圍步長5的所有可能組合。

#### Scenario: 系統性參數覆蓋
系統需要生成所有有效的KDJ參數組合：
- K週期：5, 10, 15, ..., 300
- D週期：1, 2, ..., 20
- 有效組合：D < K（確保技術合理性）

#### Acceptance Criteria
- [ ] 生成約600個有效參數組合
- [ ] 每個組合包含唯一標識符（KDJ_[K,D]格式）
- [ ] 支持參數範圍和步長的自定義配置

### ADDED Requirement: 並行處理引擎

#### Description
實現高效的並行處理架構，支持多進程同時回測不同參數組合。

#### Scenario: 大規模並行回測
當有600個策略需要回測時，系統應該利用32核並行處理能力，將工作負載均勻分配到各進程。

#### Acceptance Criteria
- [ ] 支持32核並行處理
- [ ] 每個進程處理約20個策略
- [ ] 實現負載均衡和錯誤處理機制

### ADDED Requirement: 智能結果聚合器

#### Description
聚合所有並行進程的回測結果，進行統一分析和排名。

#### Scenario: 結果統一分析
收集所有600個策略的回測結果，按多個維度進行排名和篩選：
- Sharpe比率排名
- 年化回報排名
- 最大回撤排名
- 綜合質量評分

#### Acceptance Criteria
- [ ] 聚合所有並行進程結果
- [ ] 提供多維度排名功能
- [ ] 生成Top 10策略詳細報告

### MODIFIED Requirement: 回測性能計算

#### Description
增強回測性能計算邏輯，支持放寬信號條件下的收益計算。

#### Scenario: 分層信號收益計算
當交易信號具有不同強度時（-1, -0.5, 0.5, 1），系統應該根據信號強度調整倉位大小和收益計算。

#### Acceptance Criteria
- [ ] 支持分層倉位計算（信號強度×倉位比例）
- [ ] 實現精確的收益和風險指標計算
- [ ] 與原始系統計算結果保持兼容

### MODIFIED Requirement: 風險管理系統

#### Description
增強風險管理機制，確保放寬條件下不產生過度風險策略。

#### Scenario: 風險指標監控
在回測過程中實時監控關鍵風險指標：
- 最大回撤不超過設定閾值
- 連續虧損次數限制
- 波動率控制在合理範圍

#### Acceptance Criteria
- [ ] 實現實時風險指標監控
- [ ] 提供風險預警和停止機制
- [ ] 生成詳細的風險分析報告

### MODIFIED Requirement: MB_KDJ策略保護

#### Description
確保MB_KDJ_[10,2]策略在優化過程中得到保護，性能不應下降。

#### Scenario: 基準策略監控
在運行完整參數優化時，系統應該同時測試MB_KDJ_[10,2]策略，確保其作為基準的穩定性。

#### Acceptance Criteria
- [ ] MB_KDJ_[10,2]策略在優化中必須包含
- [ ] 監控基準策略性能變化
- [ ] 提供與基準策略的對比分析
